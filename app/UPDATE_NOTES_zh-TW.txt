================================================================
  Stream Translator ROCm 更新說明
  更新日期：2026-06-12
================================================================

【本次更新】
  新增：修正 AMD 內顯位於 GPU 0 時，未選到 RX 9070 XT 的問題；現在會列舉所有 GPU 並選擇支援的 gfx1201 顯卡。
  1. 修正設定畫面顯示「已儲存」，但重新啟動後設定恢復舊值的問題。
  2. 修正主視窗或字幕視窗關閉時，舊設定可能覆蓋剛儲存設定的問題。
  3. 設定檔改用安全的原子寫入，降低寫入中斷造成 config.yaml 損壞的風險。
  4. 設定寫入失敗時，前端會正確顯示錯誤，不再誤報儲存成功。
  5. 更新前端靜態資源。
  6. 修正停止翻譯後 ASR 子程序未完整清理、VRAM 可能持續占用的問題。
     停止時現在會終止完整程序樹，並關閉模型常駐模式。
  7. 修正不支援 GPU 架構時發生 HIP invalid device function 並卡住的問題。
     程式會先檢查 PyTorch 內建 gfx 架構，不相容時自動切換 CPU 並顯示提示。
  8. ASR 執行緒錯誤會傳回介面並結束任務，不再無限等待。
  9. 日誌中的 API Key、Token 與 Webhook URL 現在會自動遮蔽。

【從舊版更新】
  建議先完整關閉舊版 Stream Translator，再進行更新。

  方法一：完整更新（建議）
  1. 備份舊版根目錄的 config.yaml。
  2. 解壓縮新版到新的資料夾。
  3. 將備份的 config.yaml 複製到新版根目錄並覆蓋預設檔案。
  4. 啟動 Stream Translator ROCm.exe。

  方法二：覆蓋舊版程式
  1. 務必先備份 config.yaml。
  2. 將新版檔案覆蓋到舊版資料夾。
  3. 不要用新版預設 config.yaml 覆蓋原本的使用者設定。

【重要提醒】
  - config.yaml 保存 API 金鑰、模型路徑、翻譯與字幕設定，更新前請備份。
  - 不建議只替換 EXE；本次更新同時包含前端與內部程式檔案。
  - 若只更新前端，必須完整替換 _internal\backend\static 資料夾。
  - 更新完成後若畫面仍為舊版，請完全關閉程式後重新啟動。
